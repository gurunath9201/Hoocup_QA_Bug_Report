# Hoocup_QA_Bug_Report
# Hoocup AI Companion App – QA Bug Hunt Report

## Overview
This repository contains the bug testing report prepared as part of a QA testing exercise for the Hoocup AI Companion mobile application. The objective of this project was to explore the application thoroughly and identify bugs across different modules using manual testing techniques.

## Application Details
App Name: Hoocup – AI Companion  
Platform Tested: Android Mobile Application  
Testing Type: Manual Testing  

## Testing Scope
The following modules were tested during this project:

- Authentication (Google Sign-In)
- Chat and Messaging
- Audio and Video Call Features
- Navigation and UI Components
- Profile and Account Settings
- Network Handling and Offline Behavior
- Performance and Responsiveness

## Testing Approach
The application was tested using exploratory testing and basic functional testing methods. Different user scenarios were executed to observe system behavior and identify issues.

Testing activities included:

- Verifying feature functionality
- Checking UI layout and responsiveness
- Testing navigation flows
- Testing offline scenarios
- Observing performance delays
- Validating input fields and forms

## Bug Reporting
A total of **35 issues** were identified and documented. Each bug entry includes:

- Feature / Module
- Bug Description
- Steps to Reproduce
- Expected Result
- Actual Result
- Severity Level

The complete bug report is available in the PDF file included in this repository.
